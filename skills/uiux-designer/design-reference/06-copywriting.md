# Design Reference: Copywriting

## Source
*Practical UI Design, 2nd Edition by Adham Dannaway*
Pages 266-293

---

## Core Principle

**Be Concise**

Every word should earn its place. More words ≠ better communication. People often skip large blocks of text.

**Goal:** Say more with fewer words. If a word can be removed without losing meaning, remove it.

### Conciseness Guidelines

1. **Avoid unnecessary words:**
   - "actually", "basically", "really", "truthfully", "quite"

2. **Omit short joining words:**
   - "a", "an", "the" (when not needed)

3. **Cut introductory phrases:**
   - "would you like to", "in order to", "when it comes to"
   - "are you sure", "there are", "it is"

4. **Use shorter words:**
   - Convey the same message with less

5. **Keep sentences under 20 words:**
   - Long sentences with commas cause focus loss

## Sentence Case

### Use Sentence Case, Not Title Case

**Sentence case:**
- Only first word + proper nouns capitalized
- "Save post for later?"

**Title case:**
- All words capitalized (except minor words)
- "Save Post for Later?"

### Why Sentence Case?

- Simpler rules
- Easier to read
- Grounded in grammar
- No confusion about capitalization

**Why avoid title case:**
- Complex, non-standardized rules
- Harder to scan (uppercase letters interrupt reading)
- Increases cognitive load

## Plain Language

### Simple = Accessible

Imagine explaining to a 6th grader unfamiliar with the topic.

**Avoid:**
- **Jargon** — Technical language for specialists
- **Slang** — Informal language for specific groups

**Prefer:**
- Short, simple words
- Conversational tone
- Contractions: "who's", "they're", "you're"

### Language Complexity

| Don't Use | Use Instead |
|-----------|------------|
| "Execute a custom domain" | "Add a custom domain" |
| "More deets" | "Learn more" |
| "Bee's knees" | Professional terms |

## Front-Load Text

### Put Important Information First

People scan from top to bottom, left to right. Key info at the front = faster comprehension.

**Before (important info at end):**
- "Subscribe to my newsletter to learn UI design"
- "Sign up today for 30% off"

**After (important info first):**
- "Learn UI design by subscribing to my newsletter"
- "30% off if you sign up today"

### Application
- Headings
- Lists
- Links
- Buttons

## Label Actions Clearly

### Use Action Verbs

Labels should describe what happens when clicked.

| Vague | Clear |
|-------|-------|
| "Click here" | "Download report" |
| "Submit" | "Save changes" |
| "More" | "View pricing plans" |

### Button Labels

- Start with action verb
- Be specific about the outcome
- Use sentence case

**Good examples:**
- "Save article"
- "Delete account"
- "Send message"

## Write for Scanning

### People Don't Read Line-by-Line

They scan. Support this behavior:

1. **Short paragraphs** — One idea per paragraph
2. **Bullet points** — For lists of 3+ items
3. **Bold key words** — Draw attention to important terms
4. **White space** — Break up dense content

### Scanning-Friendly Content

**Before:** Long paragraph with no breaks

**After:** Short paragraphs + bullet points + white space

## Error Messages

### Be Clear and Helpful

**Bad:** "An error occurred"

**Good:** "We couldn't save your changes. Please check your internet connection and try again."

### Error Message Guidelines

1. **Explain what happened** — Not just "error occurred"
2. **Suggest a solution** — Help users fix the issue
3. **Use plain language** — No technical jargon
4. **Stay calm** — Avoid alarming language

## Microcopy

### Small Text Matters

**Help text:**
- Explain field requirements
- Use examples
- Be encouraging

**Placeholder text:**
- Show expected format
- Replace with label, don't use as label

**Confirmation messages:**
- Clear + reassuring
- Explain what happens next

### Good Microcopy Examples

| Context | Text |
|---------|------|
| Form hint | "We'll never share your email" |
| Empty state | "No articles yet. Start writing!" |
| Loading | "Saving your changes..." |
| Success | "Article saved to your library" |

## Tone Guidelines

### Match Brand Voice

**Professional:**
- Clear, direct language
- No slang or humor

**Casual:**
- Contractions allowed
- Friendly, conversational

**Technical:**
- Precise terminology
- Clear instructions

### Tone Consistency

- Use same tone across all interfaces
- Match user expectations
- Consider the context (error vs success messages)

## Key Takeaways

1. **Every word earns its place** — Remove if not needed
2. **Sentence case** — Simpler and easier to scan
3. **Plain language** — Accessible to everyone
4. **Front-load** — Put important info first
5. **Action labels** — Start with clear verbs
6. **Scan-friendly** — Short paragraphs, bullets, white space
